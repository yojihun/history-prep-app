# 과학2 소화 단원 학습 웹앱 개발 및 배포 과정 요약

본 문서는 중학교 과학2(비상 임태훈) 소화 단원 대비 인터랙티브 학습 웹사이트의 개발 및 배포 전 과정을 기록한 요약 문서입니다.

---

## 1. 프로젝트 개요 및 요구사항
* **목표**: 기존에 성공적으로 구축된 역사 학습 앱의 기능적 구조(빈칸 채우기, 카드 학습, 게임, AI 주관식 채점 및 힌트 제공 등)를 그대로 계승하면서, **중학교 과학2 소화 단원**에 최적화된 새로운 학습 사이트를 별도로 구축 및 배포합니다.
* **배포 환경**: 역사 앱과 혼선이 생기지 않도록 독립된 Git 저장소를 생성하고 Vercel에 신규 독립 프로젝트로 배포합니다.

---

## 2. 세부 개발 및 가공 과정

### 2.1. 과학 교과 텍스트 추출 및 데이터 설계
* **텍스트 추출**: 파이썬 스크립트를 사용하여 제공된 `수행평가관련 - 중학_과학2_비상(임태훈).pdf` 파일에서 텍스트 데이터를 추출 및 분석했습니다.
* **단원 정보 설계 (`src/data/chapters.js`)**:
  * **01 음식물에 들어 있는 영양소**: 3대 영양소의 특징 및 영양소 검출 시험(아이오딘, 뷰렛, 베네딕트, 수단 Ⅲ) 내용 구성.
  * **02 영양소를 소화하는 소화계**: 소화관(입-식도-위-소장-대장-항문)과 소화샘(침샘, 간, 쓸개, 이자)의 명칭 및 역할 정의.
  * **03 소화효소의 작용과 흡수**: 소화효소(아밀레이스, 펩신, 트립신, 라이페이스)에 의한 영양소의 최종 분해 상태와 융털(모세혈관, 암죽관)을 통한 흡수 경로 구축.
* **퀴즈 문항 구성**: 핵심 개념을 평가하는 실전 객관식 퀴즈 11문항과 단원별 AI 서술형 평가 문항을 탑재했습니다.

### 2.2. 소화 경로 인터랙티브 타임라인 개편 (`src/components/Timeline.jsx`)
* 기존 역사 연표 형식의 타임라인을 음식물의 이동 경로인 **소화 경로 탐색 타임라인**으로 전면 개편했습니다.
* **경로 흐름**: 영양소 섭취(Ingestion) ➔ 입(Mouth) ➔ 위(Stomach) ➔ 작은창자(Small Intestine) ➔ 영양소 흡수(융털/Absorption) ➔ 큰창자 및 배설(Large Intestine/Excretion) 단계로 구성되어, 각 단계를 클릭하여 직관적인 핵심 과학 정보를 학습할 수 있습니다.

### 2.3. 에메랄드 그린 컬러 테마 적용 (`src/index.css`)
* 과학 교과의 맑고 학구적인 분위기에 매치되도록 기존의 브라운/블루 톤에서 **에메랄드 그린/네이비 다크 모드 테마**로 디자인 시스템을 변경했습니다.
* 주요 CSS 변수(`--primary`, `--secondary`, `--accent` 등)를 Emerald Green(`#059669`) 계열로 교체하여 완성도 높은 색감 조화를 제공합니다.

### 2.4. 한글 텍스트 반영 이미지 생성 및 배치 (`public/images/`)
* 앱 내에서 사용되는 주요 모식도 이미지를 이미지 생성 모델을 활용해 고화질로 커스텀 생성했습니다.
* 학습 가독성을 높이기 위해 다이어그램 내부의 핵심 명칭(예: 녹말, 단백질, 지방, 입, 위, 융털, 모세혈관, 암죽관 등)을 **모두 직관적인 한국어 레이블**로 명시해 사용자의 학습 이해도를 돕도록 배치했습니다.
  * `nutrient_test.png` (영양소 검출 반응 시험관 모식도)
  * `digestive_system.png` (인체 소화계 전체 구조 모식도)
  * `villi.png` (소장 융털 미세 구조 모식도)

### 2.5. ElevenLabs AI를 이용한 캐릭터 고음질 한국어 음성 생성 (`public/audio/`)
* 역사 앱의 마스코트였던 3종의 바나나 캐릭터(세종대왕, 이순신, 선비) 콘셉트를 유지하면서, 각각의 개성에 어울리는 ElevenLabs 명품 AI 음성을 새롭게 생성해 탑재했습니다.
  * **세종대왕 바나나**: 근엄하고 중후한 왕조 톤 음성
  * **이순신 장군 바나나**: 굳건하고 씩씩한 무장 톤 음성
  * **바나나 선비**: 차분하고 논리적인 학자 톤 음성
* 응답 상황(성공, 실패, 힌트 요구) 및 캐릭터별 성격에 맞는 한국어 TTS 오디오 파일 총 18종을 생성하여 `public/audio/`에 탑재하고 동적 재생 시스템을 구축했습니다.

## 3. Git 및 Vercel 배포 세팅

### 3.1. 독립 Git 저장소 구성
* 역사 프로젝트 폴더와의 상속 관계를 끊기 위해 `science-app` 폴더 내의 기존 메타데이터 캐시(`.git`, `.vercel`)를 초기화했습니다.
* 새 저장소 초기화 및 커밋 완료:
  ```bash
  git init
  git add .
  git commit -m "Initialize Science App & Apply updates"
  ```

### 3.2. Vercel 독립 프로젝트 생성 및 배포
* Vercel CLI의 `--scope` 옵션을 활용하여 `yojihuns-projects` 스코프 하위에 **`science-app`**이라는 별도의 독립 프로젝트를 생성하여 배포를 수행했습니다.
* **프로덕션 대표 도메인 주소**: [https://science-app-sigma.vercel.app](https://science-app-sigma.vercel.app)

### 3.3. Gemini API 키 연동
* AI 기반 서술형 답안 채점과 학습 목표 노트패드 채점을 원활히 제공하기 위해 Vercel 환경 설정에 **`VITE_GEMINI_API_KEY`** 변수를 등록하고 프로덕션 빌드와 동기화시켰습니다.
  * `Development`, `Preview`, `Production` 전 환경에 키 매핑 적용 완료.

## 4. 디버깅 및 마감 보완 사항
* **실전 퀴즈 문항 노출 오류 수정**: 퀴즈 데이터 중 첫 번째 문항에만 `chapterId` 필드가 할당되어 있어 사이트에서 1문제만 출제되던 버그를 발견하여, 전체 11문항에 `chapterId: "1"`을 추가해 11문제 전체가 온전하게 출제되도록 수정했습니다.
* **웹사이트 메타 타이틀 수정**: 브라우저 탭 이름이 이전 역사 프로젝트명인 `history-app`으로 출력되던 현상을 해결하기 위해 `index.html` 파일의 `<title>` 태그를 `중학 과학2 소화 단원 학습`으로 수정하여 배포했습니다.
